# Storytelling AI API

FastAPI + LangGraph backend for generating long-form stories, optional human approval, and optional AI-generated cover images.

## Overview

The pipeline runs in-process with async background tasks:

1. Plan story outline (`plan_story`)
2. Optional human gate (`await_approval` via LangGraph `interrupt`)
3. Start cover generation (`generate_cover`) in parallel
4. Generate chapters in parallel (`generate_chapter` fan-out)
5. Assemble final script (`assemble_story`)

`generate_cover` and chapter generation run concurrently after approval. The graph completes once both branches finish.

State and outputs are persisted in PostgreSQL. Progress is available through REST polling and SSE.

## Architecture

- `api/`: FastAPI routes and schemas
- `services/`: business orchestration layer
- `repositories/`: SQLAlchemy data access
- `graph/`: LangGraph topology
- `agents/`: planner/generator/assembler/cover graph nodes
- `pipeline_runner.py`: background pipeline runner + interrupt/resume handling
- `storage/`: database and object storage helpers

## Key Features

- Chapter-parallel generation using LangGraph `Send`
- Cover generation branch (`generate_cover`) running in parallel with chapter workers
- Human-in-the-loop approval with `interrupt()` and `Command(resume=...)`
- SSE status stream for UI progress updates
- Async SQLAlchemy repository/service separation
- Graceful cover fallback when Imagen is unavailable on current plan

## Quick Start

### Prerequisites

- Python 3.11+
- PostgreSQL
- Google AI access (Gemini Developer API or Vertex AI)

### Install

```bash
pip install -r requirements.txt
```

### Configure

Use one of the auth modes below in `.env`.

#### Option A: Gemini Developer API mode

```dotenv
USE_VERTEX_AI=false
GEMINI_API_KEY=your_gemini_developer_api_key
GEMINI_MODEL=gemini-2.5-flash-lite
IMAGEN_MODEL=imagen-4.0-generate-001
```

#### Option B: Vertex AI mode (recommended for production)

```dotenv
USE_VERTEX_AI=true
VERTEX_PROJECT_ID=your-gcp-project-id
VERTEX_LOCATION=us-central1
GEMINI_MODEL=gemini-2.5-flash-lite
IMAGEN_MODEL=imagen-4.0-generate-001
```

For Vertex mode, authenticate with ADC:

```bash
gcloud auth application-default login
```

Important: in Vertex mode, do not pass an API key to the `google.genai` client together with `project/location`. This project now enforces that automatically.

### Run API

```bash
uvicorn api.main:app --reload
```

### Run LangGraph local dev server

```bash
langgraph dev
```

### Run tests

```bash
pytest -q
```

## API Endpoints

- `POST /stories/generate` - create generation job
- `GET /stories/{story_id}` - fetch current status and outputs
- `GET /stories/{story_id}/cover` - fetch decoded cover image (`image/png`) if available
- `PATCH /stories/{story_id}/outline` - edit planned outline
- `POST /stories/{story_id}/approve` - resume paused approval flow
- `GET /stories/{story_id}/events` - SSE progress events
- `GET /stories/` - paginated history
- `DELETE /stories/{story_id}` - delete story and metadata
- `GET /health` - health check

## Cover Generation Behavior

- Cover image is generated by `agents/cover_artist.py` using Imagen.
- `cover_image` is stored in DB as base64 text.
- If Imagen is blocked by billing/plan restrictions, cover generation is skipped and the story run continues successfully.
- In that case, `cover_image` remains `null` and `/stories/{story_id}/cover` returns 404.

## Troubleshooting

- Error: `API key required for Gemini Developer API`
  - Cause: running Developer API mode without `GEMINI_API_KEY`.
  - Fix: set `USE_VERTEX_AI=false` and provide `GEMINI_API_KEY`, or switch to Vertex mode with `USE_VERTEX_AI=true` + `VERTEX_PROJECT_ID`.

- Error: `Project/location and API key are mutually exclusive`
  - Cause: Vertex client initialized with both key and `project/location`.
  - Fix: use Vertex mode with `project/location` only (ADC auth).

- Error: `Imagen ... only available on paid plans`
  - Cause: current account plan does not allow Imagen generation.
  - Fix: upgrade plan in Google AI, or keep current behavior (cover skipped, story still completes).

## Notes

- `langgraph.json` exports `compiled_graph` without a custom checkpointer for Studio/platform compatibility.
- The standalone runner compiles its own graph with durable local SQLite checkpointer/cache for local resume + cache behavior.
